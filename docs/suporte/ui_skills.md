# UI Skills — Referência Operacional

## Objetivo

Definir quando vale a pena acionar skills ou fluxos padronizados para tarefas de UI e
execução repetitiva.

---

## Quando usar

- Tarefas recorrentes com baixa variação
- Instalação de recursos ou padrões que seguem um passo a passo estável
- Auditorias rápidas de consistência visual ou estrutural

---

## Quando não usar

- Tarefas exploratórias ainda sem processo definido
- Mudanças únicas que exigem forte adaptação ao contexto
- Casos em que a skill adiciona mais complexidade do que reduz

---

## Critério de uso

Só consolidar uma skill quando o padrão já estiver estável e validado em múltiplas
execuções.
