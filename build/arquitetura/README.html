O que é esta pasta

Contém as premissas técnicas que todo projeto FXL deve seguir quando desenvolvido via Claude Code. É a fundação que garante qualidade, segurança e consistência entre projetos.

Cada documento desta pasta define um aspecto específico da construção: stack, banco de dados, deploy, segurança, testes e como montar o prompt de desenvolvimento.

Como usar

Ao iniciar um novo projeto (Fase 3), o CLAUDE.md do repositório do projeto deve referenciar esta pasta por caminho absoluto. Veja o template em claude_md_projeto.html.

Antes de começar, leia obrigatoriamente:
- [CAMINHO_ABSOLUTO]/fxl-processo-padrao/build/arquitetura/premissas_gerais.md
- [CAMINHO_ABSOLUTO]/fxl-processo-padrao/build/arquitetura/supabase.md
- [CAMINHO_ABSOLUTO]/fxl-processo-padrao/build/arquitetura/deploy_vercel.md
- [CAMINHO_ABSOLUTO]/fxl-processo-padrao/build/arquitetura/seguranca.md
- [CAMINHO_ABSOLUTO]/fxl-processo-padrao/build/arquitetura/testes.md
Substitua [CAMINHO_ABSOLUTO] pelo caminho real na sua máquina antes de usar. Ex: /Users/operador/projetos

Evolução

Cada projeto concluído deve contribuir com aprendizados para esta pasta. Se um padrão recorrente emergir em um projeto — seja uma solução elegante, um erro evitado ou uma decisão arquitetural importante — registrar aqui para beneficiar os próximos projetos.

A pasta build/ é onde o processo técnico da FXL vive e evolui. A visão de longo prazo é que cada documento aqui se torne mais completo e preciso com cada projeto.
