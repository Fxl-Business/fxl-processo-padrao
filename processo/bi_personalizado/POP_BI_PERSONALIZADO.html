Quando usar este POP

  • É para um cliente específico com dados únicos
  • Não será reutilizado diretamente para outros clientes
  • Parte de uma reunião de diagnóstico com o cliente

Exemplos: BI Tatu do Bem, BI Mercadão, BI [qualquer empresa com operação própria]

⚠️ Se for um produto FXL com vida própria, usar POP_PRODUTO.
Módulo Descrição
Financeiro Normalmente resolvido com software White Label (ex: Conta Azul). FXL raramente constrói dashboard financeiro customizado.
Comercial Dashboard exclusivo. KPIs de vendas, funil, ticket médio, performance por vendedor, produto e região.
Operacional Dashboard exclusivo. Alta variação entre clientes — requer diagnóstico mais profundo na Fase 1.

Fase 1 — Diagnóstico Estratégico

Fase 1Dia 0 — 60 a 90 minutos

Diagnóstico Estratégico

O diagnóstico é a fundação de tudo. Um briefing incompleto gera retrabalho em todas as fases.

Campos obrigatórios do Briefing

  • Nome da empresa e segmento
  • Módulos contratados
  • Sistemas utilizados — onde os dados são gerados
  • Relatórios disponíveis — nome + colunas de cada um
  • Dores principais — o que o cliente não enxerga hoje
  • KPIs prioritários declarados — em ordem de importância
  • Frequência de análise (Diária / Semanal / Mensal)
  • Maturidade analítica (Baixa / Média / Alta)
  • Quantidade e perfis de usuários
  • Prazo esperado

Erros comuns a evitar

  • Não confirmar colunas antes de propor KPIs que dependem delas
  • Assumir que o cliente sabe o que quer — sempre cruzar com a Biblioteca de KPIs
  • Avançar sem briefing validado pelo operador

Fase 2 — Wireframe Estratégico

Fase 2D+1 após reunião

Wireframe Estratégico

O wireframe define a arquitetura completa antes de qualquer linha de código. O cliente aprova antes do desenvolvimento — isso evita retrabalho caro.

O Claude gera o Blueprint textual. O operador usa como prompt no Whimsical AI para gerar o visual. O Whimsical é apresentado ao cliente para aprovação formal.

💬 Prompt — colar no Claude Project
Com base no briefing validado, gere o Blueprint completo. Para cada tela: nome, objetivo, filtros, cards de KPI, gráficos e tabelas. Inclua a tela de Inputs. Ao final, gere o Prompt Whimsical pronto para uso.
✅ Critério de Avanço: Cliente visualiza o wireframe no Whimsical e dá aprovação formal escrita. Sem aprovação, a Fase 3 não inicia.

Fase 4 — Auditoria e Validação

Fase 4Após desenvolvimento concluído

Auditoria e Validação

A auditoria é executada por um profissional da FXL. O Claude gera o checklist — a execução é humana.

Checklist base — obrigatório em todos os projetos

  • Faturamento total bate com relatório de origem (tolerância: 0%)
  • Filtro de período funciona e atualiza todos os KPIs
  • Filtro de segmento/unidade isola corretamente os dados
  • Ticket médio = Faturamento ÷ Pedidos para o período filtrado
  • Margem bruta calculada corretamente (se aplicável)
  • Top rankings ordenados — verificar 3 primeiros e últimos manualmente
  • Tela de Inputs: upload funciona e dados aparecem no dashboard
  • Skeleton loading aparece antes dos dados ao recarregar
  • Seletor de período: mês atual como padrão ao abrir
  • Todos os KPIs do briefing estão presentes — nenhum pode faltar
  • Sem erros de console / telas quebradas
💬 Prompt — colar no Claude Project
Com base no briefing deste projeto, gere o Checklist de Auditoria completo: checklist base + itens específicos dos KPIs, segmento e particularidades deste cliente.
✅ Critério de Avanço: Zero itens bloqueantes. Melhorias podem seguir para a Fase 5, mas não bloqueiam a entrega.

Fase 5 — Entrega + Validação Assistida

Fase 5Imediato após auditoria

Entrega + Validação Assistida

Sistema entregue com status "Em fase de checkagem assistida". O cliente tem 15 dias para validar, testar e solicitar ajustes. Após esse período, o produto é considerado validado.

Cláusulas contratuais obrigatórias

  • Cliente responsável pela veracidade dos dados enviados
  • Ajustes após 15 dias podem gerar aditivo contratual
  • Mudança estrutural de escopo exige nova proposta
  • O BI não substitui a contabilidade oficial
  • A entrega depende do envio correto dos relatórios
