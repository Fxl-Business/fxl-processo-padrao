Quando usar este POP

  • É um produto com vida própria, não atrelado a um cliente específico
  • Pode atender múltiplos usuários ou empresas
  • A definição do escopo parte da FXL
Subtipo Descrição Exemplos
BI de Plataforma Atrelado aos dados de um software específico de mercado BI Conta Azul, BI Mobills, BI Otimiza
BI de Segmento Agnóstico de software, construído para um vertical de mercado BI de Locação (planilha genérica)
Sistema Produto com lógica própria além de visualização Locação de Imóveis
SaaS Estágio de maturidade de qualquer produto acima Qualquer produto multi-tenant com assinatura

Fase 1 — Definição do Produto

Fase 1Interno — sem prazo de cliente

Definição do Produto

Principal diferença: não há reunião de diagnóstico com cliente. A definição parte do conhecimento da FXL.

Campos obrigatórios

  • Nome e objetivo do produto
  • Fonte de dados — sistema, relatórios, colunas, formato
  • Usuário-alvo — perfil, maturidade analítica, contexto de uso
  • Módulos e KPIs — lista completa com lógica de cálculo
  • Regras de negócio do segmento
  • Funcionalidades de configuração pelo usuário final
  • Telas do sistema com objetivo de cada uma
  • Decisões de Produto em aberto — sinalizar como [DECISÃO DE PRODUTO]
  • Restrições técnicas conhecidas
⚠️ Todas as [DECISÃO DE PRODUTO] devem ser resolvidas antes de avançar para a Fase 2.
💬 Prompt — colar no Claude Project
Vou descrever um produto FXL. Gere o Documento de Definição de Produto estruturado. Sinalize todas as [DECISÕES DE PRODUTO] em aberto e apresente as opções para cada uma.

Fase 2 — Wireframe Estratégico

Fase 2Após Fase 1 validada

Wireframe Estratégico

Idêntica ao BI Personalizado. Diferença: a aprovação é interna — não depende de cliente.

💬 Prompt — colar no Claude Project
Com base no Documento de Definição de Produto, gere o Blueprint completo. Para cada tela: nome, objetivo, filtros, cards, gráficos e tabelas. Inclua a tela de Inputs. Ao final, gere o Prompt Whimsical.
✅ Critério de Avanço: Aprovação interna da equipe FXL.

Fases 3 e 4 — Desenvolvimento e Auditoria

Idênticas ao POP de BI Personalizado. Itens adicionais na auditoria:

  • Verificar que o sistema funciona com dados de múltiplos perfis sem cruzamento
  • Para SaaS: verificar isolamento de dados entre tenants
  • Para BI de Plataforma: testar upload com arquivo real exportado do software de origem
✅ Critério de Avanço: Zero itens bloqueantes. Validação interna pela equipe FXL.

Fase 5 — Lançamento e Disponibilização

Fase 5Imediato após auditoria

Lançamento e Disponibilização

Diferença: não há período de checkagem com cliente. O produto é disponibilizado para uso.

  • Definir processo de onboarding do usuário final
  • Criar documentação de uso
  • Definir canal de suporte
  • Registrar versão de lançamento no histórico

Fase 6 — Tutorial e Documentação

Fase 6Paralelo à Fase 5

Tutorial e Documentação

Diferença: o tutorial é a documentação oficial do produto — genérico para qualquer usuário, específico para o segmento ou software.

  • O que o produto resolve e para quem
  • Como fazer upload / conectar os dados
  • Cada tela principal: KPIs, filtros, interpretação
  • Dúvidas frequentes e erros comuns

Ciclo de Evolução do Produto

  • Registrar mudanças no histórico do Documento de Definição
  • Verificar impacto em outros arquivos do repositório
  • Se escalar para SaaS: documentar arquitetura em build/arquitetura/
