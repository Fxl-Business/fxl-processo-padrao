Visão de Longo Prazo

A FXL é uma consultoria de Business Intelligence para PMEs. Hoje entregamos dashboards e sistemas que transformam dados em decisões — e decisões em resultado.

Ter um processo capaz de entender qualquer negócio e, a partir de perguntas e respostas estruturadas, gerar qualquer produto digital — dashboard, sistema, landing page, mobile ou automação — de forma progressivamente automatizada.

Todo projeto executado deve contribuir com aprendizados que aproximem o processo desse objetivo. A pasta /build do repositório é onde essa evolução técnica acontece.

Tipos de Projeto FXL

Antes de iniciar qualquer projeto, identifique o tipo. Isso determina qual POP seguir.

  • É para um cliente específico, com dados únicos do negócio? → BI Personalizado
  • É um produto com vida própria para múltiplos clientes? → Produto
Tipo Descrição Exemplos POP
BI Personalizado Construído para um cliente específico. Dados únicos, não reutilizável. BI Tatu do Bem, BI [Cliente] POP_BI_PERSONALIZADO
Produto Construído pela FXL com vida própria. Inclui BI de Plataforma, Sistema e SaaS. BI Conta Azul, Locação de Imóveis POP_PRODUTO

Fases do Processo

As fases se aplicam a todos os tipos. O que muda são as entradas e critérios — detalhados nos POPs.

Fase Nome Prazo
1 Diagnóstico Estratégico Dia 0
2 Wireframe Estratégico D+1 após reunião
3 Desenvolvimento Após aprovação do wireframe
4 Auditoria e Validação Após desenvolvimento
5 Entrega + Validação Assistida Imediato após auditoria
6 Tutorial e Mentoria Paralelo à Fase 5

Como Este Processo é Mantido

Claude Project "FXL — Processo Padrão"

Onde melhorias são discutidas e prompts de alteração são gerados.

Claude Code

Onde as alterações são executadas nos arquivos. Lê o CLAUDE.md automaticamente e sabe quais arquivos verificar em cascata.

⚠️ Uma alteração no processo geral pode refletir em múltiplos arquivos simultaneamente.

Ciclo de Evolução

  • KPIs novos → suporte/biblioteca_kpis.md
  • Problemas recorrentes na auditoria → checklist da Fase 4
  • Padrões de build → build/
  • Ajustes de UI → build/design/
  • Melhorias no briefing → processo/fases/fase1_diagnostico.md
Processo Padrão define como trabalhar → Projeto específico executa → Aprendizados voltam para o Padrão → O próximo começa mais forte
